# Cleanup Folder

Скилл для анализа папок и генерации безопасных команд очистки, которые перемещают файлы в Корзину (не удаляют безвозвратно).

## Как это работает

Песочница Claude не может напрямую запускать macOS-команду `osascript`, поэтому скилл работает в два этапа:
1. **Claude анализирует** папку через доступ к файлам — находит дубликаты и установщики
2. **Claude генерирует** готовые `osascript`-команды и сохраняет их в текстовый файл, который пользователь запускает через Terminal

Подход безопасный: всё уходит в Корзину, ничего не удаляется навсегда. Пользователь всегда может восстановить файлы.

## Требование: доступ Terminal к диску

**Каждый раз при запуске скилла напомни пользователю:** для выполнения команд очистки Terminal должен иметь полный доступ к диску. Без этого macOS заблокирует чтение файла с командами.

Как включить (одноразовая настройка):
1. Открыть **Системные настройки** (System Settings)
2. Перейти в **Конфиденциальность и безопасность** (Privacy & Security)
3. Выбрать **Доступ к диску** (Full Disk Access)
4. Включить тумблер напротив **Терминал** (Terminal)

Если Terminal нет в списке — нажать «+» и добавить его из `/Applications/Utilities/Terminal.app`.

## Пошаговый порядок работы

### 1. Определить целевую папку

**Обязательно спроси у пользователя, какую папку нужно почистить** — даже если кажется очевидным (например, что это Downloads). Не делай предположений, всегда уточняй. Используй `AskUserQuestion` для этого.

После того как пользователь указал папку, убедись, что есть доступ к ней (при необходимости используй `request_cowork_directory`).

Определи **реальный macOS-путь** к папке. Примонтированный путь в песочнице (например, `/sessions/.../mnt/Downloads`) соответствует реальному пути на маке пользователя (например, `/Users/<username>/Downloads`). Для osascript-команд нужен именно реальный macOS-путь. Его можно определить по структуре монтирования или уточнить у пользователя.

### 2. Запустить анализ

Запусти скрипт анализа:

```bash
python3 <путь-к-скиллу>/scripts/analyze_folder.py <путь-к-папке>
```

Скрипт выводит JSON-отчёт в stdout с разделами: `duplicates`, `installers` (раздел `app_bundles` игнорируй — .app-пакеты не удаляем, они могут быть рабочими приложениями).

Как скрипт находит дубликаты:
- Считает MD5-хеши всех файлов
- Находит группы файлов с одинаковыми хешами
- В каждой группе выделяет файлы с суффиксами копий — `(1)`, `(2)`, `(3)` и т.д.
- Помечает файл на удаление только если **оригинал** (без суффикса) существует и имеет тот же хеш
- Это консервативный подход: если кто-то переименовал файл и оригинал пропал, копия не будет помечена

### 3. Показать отчёт

Покажи пользователю понятную сводку:
- **Дубликаты**: сколько найдено, общий размер, характерные примеры
- **Установщики** (DMG/PKG): список с размерами
- **Итого можно освободить**

**Важно: НЕ предлагай удалять .app-пакеты.** Приложения могут лежать в Downloads не по ошибке — пользователь может намеренно запускать их оттуда. Удаление .app может сломать рабочие программы.

Не генерируй команды сразу — дай пользователю решить, что именно удалять.

### 4. Сгенерировать osascript-команды и сохранить в файл

На основе выбора пользователя сгенерируй `osascript`-команды и **запиши их в текстовый файл** в целевой папке пользователя. Не выводи команды в чат — при большом количестве файлов это неудобно и может сломать вывод.

Имя файла должно содержать дату и время, чтобы не перезаписывать предыдущие файлы команд: `cleanup_commands_YYYY-MM-DD_HH-MM.txt` (например, `cleanup_commands_2026-04-03_15-30.txt`). Это важно, потому что пользователь мог ещё не выполнить команды из предыдущего файла.

Формат команд:

```
osascript -e 'tell app "Finder" to delete POSIX file "<полный-macOS-путь>"'
```

После сохранения файла дай пользователю команду для запуска в Terminal:

```bash
while IFS= read -r cmd; do eval "$cmd"; done < ~/Downloads/cleanup_commands_YYYY-MM-DD_HH-MM.txt
```

(подставь актуальное имя файла)

Эта конструкция читает файл построчно и выполняет каждую команду по очереди. Не зависнет даже на тысячах файлов.

Важные моменты:
- Каждая `osascript`-команда должна быть на отдельной строке
- Если файла уже нет, команда выдаст ошибку, но ничего не сломает
- Не используй `mv ~/.Trash/` — у Terminal часто нет прав на запись в `.Trash`
- Не записывай команды в .sh-файл — macOS-карантин блокирует запуск скриптов, созданных из песочницы. Обычный .txt-файл с `while read` обходит это ограничение

### 5. После выполнения команд

Если пользователь сообщает о результатах, проверь что получилось:
- Перезапусти анализ, чтобы убедиться, что файлы удалены
- Сообщи, сколько места освободилось
- Предложи дополнительные варианты очистки, если они есть
